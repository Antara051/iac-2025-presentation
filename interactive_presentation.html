The Challenge: A Reliable Link

Small satellites like VERTECS have huge scientific ambitions. But without a robust, intuitive, and reliable link to the ground, they're flying blind.

  • ✖ Commercial software is often too expensive and inflexible for academic missions.
  • ✖ Basic scripts lack critical features like real-time visualization and structured logging.
  • ✔ Our Goal: Create a custom, operator-friendly Ground Station Software to empower the VERTECS mission.

The VERTECS 6U Nanosatellite for observing Extragalactic Background Light.

How it's Built

The development followed a hybrid Agile-V Model, combining the flexibility of Agile with the rigorous verification of the traditional V-Model—perfect for academic nanosatellite projects.

Connection Layer

Manages the raw data link between the satellite and the ground station, ensuring reliable communication over the S-band connection.

Uplink Layer

Formats operator commands into mission-compliant packets, adding timestamps and redundancy checks to ensure every command is traceable and verifiable.

Downlink Layer

Extracts, filters, and reconstructs housekeeping data from the raw telemetry stream, preparing it for real-time analysis.

Visualization Layer

Transforms technical parameters into intuitive visuals: LED indicators for system health, summary panels, and time-series plots for trend analysis.

Persistence Layer

Records every received packet, command, and decoded parameter into a four-tier logging system for post-mission analysis. Nothing is lost.
